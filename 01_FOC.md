## 四、FOC 整体控制链路，一定要能从头顺下来

### 1. 整体流程图

```text
目标转矩 / 目标速度
        ↓
外环（可选：位置环 / 速度环）
        ↓
给出 iq_ref，通常 id_ref = 0
        ↓
ADC 采样两相电流 ia、ib，计算出ic
        ↓
Clarke：abc -> αβ 求得定子磁场
        ↓
Park：αβ -> dq 通过电角度和极对数求得转子磁场
        ↓
电流环 PI：id、iq 跟踪 id_ref、iq_ref 
        ↓
得到 ud、uq
        ↓
反 Park：dq -> αβ
        ↓
SVPWM：Uα、Uβ -> 先反克拉克，求扇区，计算有效矢量和零矢量作用时间，得到三相占空比 
        ↓
HRTIM 输出互补 PWM
        ↓
逆变桥驱动三相电机
        ↓
编码器反馈机械角 -> 电角度 -> 回到控制器
```

### 2. 你要能一口气讲下来

> 整个 FOC 的主链路是：先根据编码器得到转子位置，换算出电角度；再在 PWM 周期中间采样两相电流，同时计算出第三相的电流。通过 Clarke 变换得到静止坐标系下的 `αβ` 电流，再通过 Park 变换投影到和转子磁场同步旋转的 `dq` 坐标系。这样 `id` 表示励磁分量、`iq` 表示转矩分量。然后分别和参考值比较，经过 PI 电流环得到 `ud`、`uq`，再反变换回 `αβ` 电压矢量，通过 `SVPWM` 生成三相 PWM 占空比，最后驱动逆变桥输出三相电压控制电机。

## 六、为什么要做 Clarke / Park 变换

### 1. Clarke 变换等幅值形式

把三相静止坐标系 `abc` 变成两相静止坐标系 `αβ`。

核心目的：

- 三相量彼此耦合，不好控制
- 三相平衡系统满足 `ia + ib + ic = 0`
- 所以三相里其实只有两个独立变量
- 先降维到 `αβ` 会更方便后续处理

常见表达：

```text
abc -> αβ
```
![[附件/Pasted image 20260605144950.png|239]]
![[附件/Pasted image 20260605145143.png|384]]
![[附件/Pasted image 20260605145235.png|384]]

### 2. Park 变换做了什么

把定子坐标系`αβ` 变成同步转子旋转坐标系 `dq`。

核心目的：

- 把交流量变成“近似直流量”
- 在同步旋转坐标系里，控制目标更直观
- `id` 和 `iq` 分别对应磁链与转矩
![[附件/Pasted image 20260605145850.png|343]]

常见表达：

```text
αβ -> dq
```

### 3. 面试回答版

> Clarke 变换是把三相系统投影到两相静止坐标系，降低维度；Park 变换是再把静止坐标系旋转到和转子磁场同步的坐标系里。这样原来随时间变化的三相正弦量，在 `dq` 坐标系里会变成更适合 PI 控制的直流量，从而实现励磁和转矩的解耦。

## 七、为什么一般 `id = 0`，`iq` 控制转矩

### 1. 含义

- `id`：与转子磁链同轴的分量
- `iq`：与转子磁链正交的分量

### 2. 为什么很多场景取 `id_ref = 0`

对于表贴式 PMSM 常见控制方式：

- 永磁体本身已经提供励磁
- 不需要再额外注入 `id`
- 把 `id` 压到 0，可以减少无效电流
- 让更多电流分配给 `iq`，提高转矩利用率

### 3. `iq` 为什么决定转矩

因为电磁转矩主要与磁链和正交电流分量相关，所以：

- `iq` 增大，转矩增大

## 八、电角度和机械角度一定要讲明白

这是面试高频坑点。
### 1. 定义

- **机械角度**：转轴物理上转过的角度
- **电角度**：磁场旋转对应的角度

### 2. 关系

```text
电角度 = 机械角度 × 极对数
```

如果电机有 `p` 对极：

- 转轴机械上转一圈 `360°`
- 电角度转 `p × 360°`

### 3. 为什么 FOC 必须知道电角度

因为 Park 变换的前提就是：

- 你要知道当前转子磁场朝向
- 否则 `d/q` 坐标系就对不准磁场
- 解耦控制就会失效

### 4. 零位校准要点

面试时至少知道这几件事：

- 编码器读出来通常是机械角
- 要先乘极对数得到电角度
- 还要减去机械安装误差对应的零位偏移
- 这个零位往往需要上电标定或出厂标定

### 5. 面试回答版

> FOC 实际使用的是电角度，不是机械角度。因为坐标变换需要和转子磁场严格对齐，而磁场周期和极对数相关，所以电角度等于机械角度乘极对数。另外编码器安装会有零位偏差，所以还要做零位校准，否则 `dq` 坐标轴对不准，电流环性能会明显变差，严重时会抖动甚至无法闭环。

## 九、结合我的项目：M3508 内置 SinCos 编码器怎么给 FOC 提供角度

这是我这个项目里一个很关键的实现细节。

### 1. 它不是 SPI 数字角度编码器

是输出两路模拟正余弦信号：

```text
SIN = sinθ
COS = cosθ
```

也就是说，编码器本身只是给出和转子位置相关的模拟量，真正的角度计算是在 MCU 侧完成的。

### 2. 整体链路

在我的代码里，角度获取链路可以概括成：

```text
转子转动
↓
编码器输出 SIN / COS 模拟电压
↓
ADC 同步采样两路信号
↓
减去中心值 offset
↓
除以半波幅，做归一化
↓
atan2(SIN, COS)
↓
得到当前角度
↓
减去电角度零偏
↓
送给 Park 变换
```

### 3. 为什么不能直接 `atan2(raw_sin, raw_cos)`

因为实际硬件信号不是理想的：

- 不是以 `0V` 为中心
- `SIN` 和 `COS` 的中心值可能不同
- 两路幅值也可能不同

所以如果直接拿原始 ADC 电压去做：

```c
theta = atan2(raw_sin, raw_cos);
```

那么算出来的轨迹会从圆变成椭圆，角度会有明显误差。

因此必须先做：

```c
sin_norm = (sin_v - sin_offset) / sin_amp;
cos_norm = (cos_v - cos_offset) / cos_amp;
theta = atan2f(sin_norm, cos_norm);
```


### 5. 这个角度在我的代码里就是电角度

M3508 这个项目里，代码按 `7` 对极处理，也就是：

```text
电角度 : 机械角度 = 7 : 1
```

如果编码器先给的是一机械圈 `0 ~ 2π` 的机械角，那么才需要：

```text
θe = 7 × θm - θoffset
```

但我这个 M3508 SinCos 编码器输出的正余弦本身就是按电周期变化的，所以 `atan2(SIN, COS)` 算出来的是一圈电周期内的相位角。代码里也是直接把它赋给 `angle_el`，减去电角度零偏后送入 Park 变换：

```text
θe = θsincos - θoffset
```

所以在我的项目里，不是先拿机械角再乘 `7` 得到电角度，而是直接拿 SinCos 解算出来的电角度给 FOC 使用。

### 6. 机械角是怎么得到的

机械角反而是后面通过电角度增量反推出来的：

```text
Δθm = Δθe / 7
```

也就是说：

- FOC 真正控制时优先用的是电角度
- 机械角更多是为了位置、显示、测速等用途再反推出来

### 7. 零位校准为什么还是必须做

即使已经通过 `atan2(SIN, COS)` 求出了角度，也还不能直接拿去做 Park 变换，因为：

- 这个角度是“传感器坐标系里的角度”
- 不一定正好对应定子 `α` 轴的零点

所以还必须减去电角度零偏：

```text
θpark = θsincos - θoffset
```

1. 先不给速度闭环，只输出一个固定方向的电压矢量，比如 `Ud > 0, Uq = 0`
2. 这个固定磁场会把转子吸到一个已知的电角度方向
3. 等电机稳定后，读取当前 `atan2(SIN, COS)` 出来的角度
4. 这个角度就是传感器坐标系和定子 `α` 轴之间的偏差
5. 把它作为 `angle_ele_off` 保存，后续运行时用 `θpark = θsincos - θoffset`

我当前代码里 `angle_ele_off = -0.35f` 是调试后写死的值。工程化做法是每台电机标定一次，然后存到 Flash 或 EEPROM，上电后直接读取。

### 8. 每台电机都要校准吗

> FOC 的电角度零偏一般要按电机个体标定，因为它和编码器安装位置、转子磁场零点、定子 `α` 轴以及接线方向都有关系。我项目里调试阶段是把 `angle_ele_off` 写成固定值，工程化做法会通过 D 轴注入或开环对齐测出零偏，并存到非易失存储器里，之后上电直接读取使用。

### 9. 面试回答版

> 我这个项目里的 M3508 不是通过 SPI 直接读取数字角度，而是内置 SinCos 编码器，输出两路模拟正余弦信号。MCU 用 ADC 采集 `SIN/COS` 两路电压后，先做中心值补偿和幅值归一化，再通过 `atan2` 计算角度。这个 SinCos 信号本身按电周期变化，所以代码里把 `atan2` 得到的角度直接作为电角度 `angle_el` 使用，减去电角度零偏后送入 Park 变换。由于电机有 7 对极，机械角是后面通过电角度增量除以 7 反推出来的。

### 10. 我的 G474 里 ADC 和 PWM 是怎么配的

我这个项目用的是 `STM32G474` 的 `HRTIM + ADC + DMA`：

- `HRTIM` 输出三相互补 PWM：`TA1/TA2`、`TB1/TB2`、`TD1/TD2` 对应三组半桥
- 开了死区：防止同一半桥上下 MOS 同时导通
- `CMP1` 用来产生 PWM 占空比
- `CMP3` 用来产生 `HRTIM_TRG3`，触发 ADC 采样
- `ADC1` 采电流：`Ib`、`Ia`、`Iall`
- `ADC2` 采编码器：`COS`、`SIN`
- 两个 ADC 都是 `HRTIM_TRG3` 上升沿触发，并用 `DMA` 搬到 buffer

`HRTIM` 这里选 `MUL8`，是为了用 G474 的高分辨率 PWM 能力。系统时钟是 `170MHz`，HRTIM 等效计数频率约为：

```text
170MHz × 8 = 1.36GHz
```

`Period = 40000` 时，PWM 频率约为：

```text
1.36GHz / 40000 ≈ 34kHz
```

选择这个量级，是在可听噪声、开关损耗、占空比分辨率和 ADC 采样窗口之间折中。频率再高，开关损耗和发热会上升，ADC 采样窗口也会变短；频率太低，电流纹波和噪声会更明显。

死区也需要写参数。我代码里 rising/falling deadtime 都是 `300`，按 `1.36GHz` 粗算约：

```text
300 / 1.36GHz ≈ 220ns
```

死区的作用是避免同一半桥上下 MOS 同时导通。死区太小有直通风险，死区太大又会带来输出电压畸变、电流波形失真和低速转矩变差。

这样配置的目的不是“能采到就行”，而是让采样时刻和 PWM 时序绑定。电流采样如果靠软件随便读，很容易采到开关瞬间的噪声；用 HRTIM 在固定位置触发 ADC，可以尽量避开开关边沿，让电流和 SinCos 采样更稳定。

我这里 ADC 是 `12 bit + 16x oversampling + no right shift`，所以 DMA 里的值接近：

```text
4095 × 16 = 65520
```

因此代码里用 `65535` 把 ADC 值换算成电压。

面试时可以这样说：

> 我项目里用 `STM32G474` 的 HRTIM 产生三相互补 PWM，并配置死区保护三相桥。HRTIM 选 `MUL8`，`Period=40000`，所以 PWM 约 `34kHz`；死区 rising/falling 都设为 `300`，约 `220ns`。`CMP1` 负责 PWM 占空比，`CMP3` 负责在固定 PWM 时刻触发 ADC。ADC1 采两相电流和总电流，ADC2 采 SinCos 编码器的 `SIN/COS`，两个 ADC 都通过 `HRTIM_TRG3` 上升沿触发并用 DMA 搬运。这样可以让采样和 PWM 同步，减少软件轮询抖动和开关噪声对电流环的影响。

## 十、三相电流为什么通常只采两相

### 1. 原因

三相平衡系统里：

```text
ia + ib + ic = 0
```

所以：

- 已知 `ia`、`ib`
- 就能推出 `ic = -(ia + ib)`

### 2. 工程好处

- 少一个采样通道
- 降低硬件复杂度
- 降低成本

### 3. 面试回答版

> 在平衡三相系统里，三相电流和为 0，所以工程上通常采两相电流，第三相通过约束关系计算出来。这样能减少 ADC 通道和采样电路复杂度，同时不影响 FOC 的电流重构。

## 十一、为什么 ADC 采样一般放在 PWM 中点

这是你项目实现细节里很值得讲的一点。

### 1. 原因

逆变器在开关瞬间会有：

- 死区影响
- 开关噪声
- 电流纹波
- 功率器件导通瞬态干扰

如果在开关边沿附近采样：

- 噪声大
- 电流不稳定
- 采样值误差更明显

### 2. 为什么中点更好

通常中心对齐 PWM 中间时刻：

- 开关状态相对稳定
- 电流波形更平滑
- 更适合做相电流采样

### 3. 你项目怎么讲

> 我项目里是用 `STM32G4` 的定时器事件去同步触发 ADC 采样，并通过 `DMA` 搬运数据。采样点选择的核心思路，是尽量避开 PWM 翻转附近的噪声区，通常放在 PWM 周期中相对稳定的位置，这样更有利于电流环闭环控制。

### 4. 面试回答版

> 电流采样一般放在 PWM 中点附近，是因为这个时刻功率管状态更稳定，采样更容易避开开关瞬态、死区和噪声干扰。这样采到的相电流更接近真实值，电流环更稳定。

## 十二、为什么用 `ADC + DMA`

### 1. 不直接 CPU 轮询的原因

- FOC 控制频率高
- 采样时刻要求准
- CPU 轮询会带来抖动
- 占用 CPU 资源多

### 2. `DMA` 的好处

- 减少 CPU 搬运数据开销
- 保证采样链路更稳定
- 提高实时性
- 让 MCU 资源留给控制计算和通信任务

### 3. 你项目怎么讲

> 在高频电流环里，采样时序很关键，所以我用定时器触发 ADC，再用 DMA 自动搬运采样结果。这样可以减少软件轮询带来的时延和抖动，让电流采样与 PWM 保持严格同步。

## 十三、HRTIM / 高级定时器在 FOC 里干什么

### 1. 核心用途

- 产生高频 PWM
- 生成三相互补 PWM
- 插入死区时间
- 同步触发 ADC
- 触发保护关断

### 2. 为什么重要

电机控制里对 PWM 的要求通常是：

- 频率高
- 分辨率高
- 相位关系准
- 保护响应快

`STM32G4` 这类带电机控制特性的 MCU，就是为了这种场景设计的。

### 3. 面试回答版

> 在我的项目里，`HRTIM` 主要负责高频高精度 PWM 输出，同时生成互补波形、设置死区，并与 ADC 采样事件同步。对于 FOC 来说，PWM 时序、采样时序和保护动作都很关键，所以定时器不是简单“出波”，而是整个控制时序的核心。

### 1. 电流环的任务

- 让 `id` 跟踪 `id_ref`
- 让 `iq` 跟踪 `iq_ref`

### 2. 为什么用 PI

因为在 `dq` 坐标系里：

- PI 控制简单且有效

## 十六、SVPWM 是干什么的

### 1. 本质

`SVPWM` 的目标是：

- 根据目标电压矢量
- 合成三相逆变器的开关时序
- 最终得到三相 PWM 占空比
![[附件/Pasted image 20260605195016.png]]
### 2. 为什么不用简单 SPWM

相对 SPWM，SVPWM 常见优势：

- 直流母线利用率更高
- 输出能力更强
- 谐波性能更好
### 3. 你怎么讲比较稳

> 电流环输出的是 `ud/uq`，经过反 Park 变换会得到静止坐标系下的目标电压矢量 `Uα/Uβ`。`SVPWM` 的任务就是根据这个目标电压矢量所在扇区和作用时间，计算出三相桥的占空比，并交给定时器输出。
### 4. 面试回答版

> `SVPWM` 本质上是把目标电压矢量映射成三相逆变器的开关时间分配。相比普通 SPWM，SVPWM 的母线电压利用率更高，所以在电机控制里很常见。

## 十七、项目实现视角下，我的 FOC 控制器可以怎么讲

这一段要能直接复述。
### 1. 项目版本

> 我在项目里做的是一个基于 `STM32G4` 的有感 FOC 控制器。硬件侧重点有两个：一是利用 `HRTIM` 产生高频互补 PWM，并配合死区和同步触发；二是利用 `ADC + DMA` 做相电流同步采样，尽量减少采样抖动和 CPU 开销。控制算法链路上，我实现了 `Clarke/Park` 变换、`dq` 电流闭环 PI，以及 `SVPWM` 调制。位置反馈采用编码器角度，先从机械角换算到电角度，再参与 Park 变换。整个系统最终实现了 `M3508` 无刷电机的闭环驱动控制。

### 2. 如果面试官追问“你具体做了什么”

> 我的工作不只是把公式搬上去，而是把控制链路真正跑起来，包括 PWM 和 ADC 的时序配合、两相电流采样、角度换算、闭环调试，以及让电机从能转到稳定闭环。实际调试时我会重点看角度是否对齐、电流采样是否稳定、电流环有没有振荡，以及占空比输出和实际转速响应是否一致。

## 二十、3 分钟项目回答模板

> 我做过一个基于 `STM32G4` 的有感 FOC 电机矢量控制器，用在灌溉水泵相关场景。这个项目里，我主要做了三部分。
>
> 第一部分是底层时序和驱动。利用 `STM32G4` 的 `HRTIM` 输出高频互补 PWM，并结合定时器事件同步触发 ADC，在 PWM 合适时刻做两相电流采样，再通过 `DMA` 自动搬运数据，降低 CPU 开销并保证采样实时性。
>
> 第二部分是控制算法链路。我构建了从相电流采样到 `Clarke/Park` 变换，再到 `dq` 电流 PI 闭环和 `SVPWM` 调制的整套 FOC 链路。位置反馈侧使用编码器角度，先做机械角到电角度换算，再参与坐标变换，实现 `id/iq` 解耦控制。
>
> 第三部分是闭环调试与工程落地。调试时我重点关注角度零位是否校准、电流采样时刻是否稳定、电流环参数是否合适，以及是否存在过流、抖动、响应不一致等问题。最终实现了 `M3508` 无刷电机的稳定闭环驱动。
>
> 这个项目让我对 FOC 不只是停留在公式层面，而是对 PWM、采样、角度反馈、闭环控制和保护逻辑之间的配合有了更工程化的理解。
