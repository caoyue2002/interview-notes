
# 1 FOC 整体控制链路

## 1.1 整体流程图

```text
目标转矩 / 目标速度
        ↓
外环（可选：位置环 / 速度环）
        ↓
给出 iq_ref，通常 id_ref = 0
        ↓
ADC 采样两相电流 ia、ib，计算出ic
        ↓
Clarke：abc -> αβ 求得定子磁场
        ↓
Park：αβ -> dq 通过电角度和极对数求得转子磁场
        ↓
电流环 PI：id、iq 跟踪 id_ref、iq_ref 
        ↓
得到 ud、uq
        ↓
反 Park：dq -> αβ
        ↓
SVPWM：Uα、Uβ -> 先反克拉克，求扇区，计算有效矢量和零矢量作用时间，得到三相占空比 
        ↓
HRTIM 输出互补 PWM
        ↓
逆变桥驱动三相电机
        ↓
编码器反馈机械角 -> 电角度 -> 回到控制器
```


> 整个 FOC 的主链路是：先根据编码器得到转子位置，换算出电角度；再在 PWM 周期中间采样两相电流，同时计算出第三相的电流。通过 Clarke 变换得到静止坐标系下的 `αβ` 电流，再通过 Park 变换投影到和转子磁场同步旋转的 `dq` 坐标系。这样 `id` 表示励磁分量、`iq` 表示转矩分量。然后分别和参考值比较，经过 PI 电流环得到 `ud`、`uq`，再反变换回 `αβ` 电压矢量，通过 `SVPWM` 生成三相 PWM 占空比，最后驱动逆变桥输出三相电压控制电机。

# 2 Clarke / Park 变换

## 2.1 Clarke 变换等幅值形式

把三相静止坐标系 `abc` 变成两相静止坐标系 `αβ`。

核心目的：

- 三相量彼此耦合，不好控制
- 三相平衡系统满足 `ia + ib + ic = 0`
- 所以三相里其实只有两个独立变量
- 先降维到 `αβ` 会更方便后续处理

常见表达：

```text
abc -> αβ
```
![[附件/Pasted image 20260605144950.png|239]]
![[附件/Pasted image 20260605145143.png|384]]
![[附件/Pasted image 20260605145235.png|384]]

## 2.2 Park 变换做了什么

把定子坐标系`αβ` 变成同步转子旋转坐标系 `dq`。

核心目的：

- 把交流量变成“近似直流量”
- 在同步旋转坐标系里，控制目标更直观
- `id` 和 `iq` 分别对应磁链与转矩
![[附件/Pasted image 20260605145850.png|343]]

常见表达：

```text
αβ -> dq
```

# 3 为什么一般 `id = 0`，`iq` 控制转矩

## 3.1 含义

- `id`：与转子磁链同轴的分量
- `iq`：与转子磁链正交的分量

对于表贴式 PMSM 常见控制方式：

- 永磁体本身已经提供励磁
- 不需要再额外注入 `id`
- 把 `id` 压到 0，可以减少无效电流
- 让更多电流分配给 `iq`，提高转矩利用率

## 3.2 `iq` 为什么决定转矩

因为电磁转矩主要与磁链和正交电流分量相关，所以：

- `iq` 增大，转矩增大

# 4 电角度和机械角度一定要讲明白

- **机械角度**：转轴物理上转过的角度
- **电角度**：磁场旋转对应的角度

## 4.1 关系

```text
电角度 = 机械角度 × 极对数
```

如果电机有 `p` 对极：

- 转轴机械上转一圈 `360°`
- 电角度转 `p × 360°`

## 4.2 为什么 FOC 必须知道电角度

因为 Park 变换的前提就是：

- 你要知道当前转子磁场朝向
- 否则 `d/q` 坐标系就对不准磁场
- 解耦控制就会失效

## 4.3 零位校准要点

面试时至少知道这几件事：

- 编码器读出来通常是机械角
- 要先乘极对数得到电角度
- 还要减去机械安装误差对应的零位偏移
- 这个零位往往需要上电标定或出厂标定

# 5 M3508 内置编码器

输出两路模拟正余弦信号：

```text
SIN = sinθ
COS = cosθ
```

| 通道    | 电压范围              | 中值           | 半幅     |
| ----- | ----------------- | ------------ | ------ |
| `SIN` | **0.29V ~ 1.76V** | **约 1.025V** | 0.735V |
| `COS` | **0.46V ~ 1.59V** | **约 1.025V** | 0.565V |

## 5.1 整体链路

角度获取链路可以概括成：

```text
转子转动
↓
编码器输出 SIN / COS 模拟电压
↓
ADC 同步采样两路信号
↓
减去中心值 offset
↓
除以半波幅，做归一化
↓
atan2(SIN, COS)
↓
得到当前角度
↓
减去电角度零偏
↓
送给 Park 变换
```

## 5.2 为什么不能直接 `atan2(raw_sin, raw_cos)`

因为实际硬件信号不是理想的：

- 不是以 `0V` 为中心
- `SIN` 和 `COS` 的中心值可能不同
- 两路幅值也可能不同

所以如果直接拿原始 ADC 电压去做：

```c
theta = atan2(raw_sin, raw_cos);
```

那么算出来的轨迹会从圆变成椭圆，角度会有明显误差。

因此必须先做：

```c
sin_norm = (sin_v - sin_offset) / sin_amp;
cos_norm = (cos_v - cos_offset) / cos_amp;
theta = atan2f(sin_norm, cos_norm);
```

M3508 这个项目里，代码按 `7` 对极处理，也就是：

```text
电角度 : 机械角度 = 7 : 1
```

如果编码器先给的是一机械圈 `0 ~ 2π` 的机械角，那么才需要：

```text
θe = 7 × θm - θoffset
```

但我这个 M3508 SinCos 编码器输出的正余弦本身就是按电周期变化的，所以 `atan2(SIN, COS)` 算出来的是一圈电周期内的相位角。代码里也是直接把它赋给 `angle_el`，减去电角度零偏后送入 Park 变换：

```text
θe = θsincos - θoffset
```

所以在我的项目里，不是先拿机械角再乘 `7` 得到电角度，而是直接拿 SinCos 解算出来的电角度给 FOC 使用。

## 5.3 机械角是怎么得到的

机械角反而是后面通过电角度增量反推出来的：

```text
Δθm = Δθe / 7
```

也就是说：

- FOC 真正控制时优先用的是电角度
- 机械角更多是为了位置、显示、测速等用途再反推出来

## 5.4 零位校准为什么还是必须做

即使已经通过 `atan2(SIN, COS)` 求出了角度，也还不能直接拿去做 Park 变换，因为：

- 这个角度是“传感器坐标系里的角度”
- 不一定正好对应定子 `α` 轴的零点

所以还必须减去电角度零偏：

```text
θpark = θsincos - θoffset
```

1. 先不给速度闭环，只输出一个固定方向的电压矢量，比如 `Ud > 0, Uq = 0`
2. 这个固定磁场会把转子吸到一个已知的电角度方向
3. 等电机稳定后，读取当前 `atan2(SIN, COS)` 出来的角度
4. 这个角度就是传感器坐标系和定子 `α` 轴之间的偏差
5. 把它作为 `angle_ele_off` 保存，后续运行时用 `θpark = θsincos - θoffset`

我当前代码里 `angle_ele_off = -0.35f` 是调试后写死的值。工程化做法是每台电机标定一次，然后存到 Flash 或 EEPROM，上电后直接读取。

# 7 为什么 ADC 采样一般放在 PWM 中点

这是你项目实现细节里很值得讲的一点。

## 7.1 原因

逆变器在开关瞬间会有：

- 死区影响
- 开关噪声
- 电流纹波
- 功率器件导通瞬态干扰

如果在开关边沿附近采样：

- 噪声大
- 电流不稳定
- 采样值误差更明显

## 7.2 为什么中点更好

通常中心对齐 PWM 中间时刻：

- 开关状态相对稳定
- 电流波形更平滑
- 更适合做相电流采样

# 8 十二、为什么用 `ADC + DMA`

## 8.1 不直接 CPU 轮询的原因

- FOC 控制频率高
- 采样时刻要求准
- CPU 轮询会带来抖动
- 占用 CPU 资源多

## 8.2 `DMA` 的好处

- 减少 CPU 搬运数据开销
- 保证采样链路更稳定
- 提高实时性
- 让 MCU 资源留给控制计算和通信任务

## 8.3 你项目怎么讲

> 在高频电流环里，采样时序很关键，所以我用定时器触发 ADC，再用 DMA 自动搬运采样结果。这样可以减少软件轮询带来的时延和抖动，让电流采样与 PWM 保持严格同步。

# 9 十三、HRTIM / 高级定时器在 FOC 里干什么

## 9.1 核心用途

- 产生高频 PWM
- 生成三相互补 PWM
- 插入死区时间
- 同步触发 ADC
- 触发保护关断

## 9.2 为什么重要

电机控制里对 PWM 的要求通常是：

- 频率高
- 分辨率高
- 相位关系准
- 保护响应快

`STM32G4` 这类带电机控制特性的 MCU，就是为了这种场景设计的。

# 11 十六、SVPWM 是干什么的

## 11.1 本质

`SVPWM` 的目标是：

- 根据目标电压矢量
- 合成三相逆变器的开关时序
- 最终得到三相 PWM 占空比
![[附件/Pasted image 20260605195016.png]]
## 11.2 为什么不用简单 SPWM

相对 SPWM，SVPWM 常见优势：

- 直流母线利用率更高
- 输出能力更强
- 谐波性能更好
## 11.3 怎么实现的

SVPWM 的输入不是直接的三相电压，而是电流环输出的 `ud / uq` 先经过反 Park 得到 `Uα / Uβ`，再经过反 Clarke / 三相投影得到 `Uu / Uv / Uw` 这三个判断量。实现时大体分成四步：

```text
1. 根据 Uu / Uv / Uw 的正负判断目标电压矢量落在哪个扇区
2. 根据扇区选择相邻两个有效电压矢量
3. 计算两个有效矢量时间 T1、T2，以及零矢量时间 T0
4. 把 T1/T2/T0 换算成三相 PWM 占空比
```

我的代码里求扇区用的是三个判断量的符号位编码：

```c
sector = (Uu > 0.0f)
       + ((Uv > 0.0f) << 1)
       + ((Uw > 0.0f) << 2);
```

等价于：

```text
sector = 1*(Uu > 0) + 2*(Uv > 0) + 4*(Uw > 0)
```

其中：

```text
Uu > 0  → 加 1
Uv > 0  → 加 2
Uw > 0  → 加 4
```

这就是 4-2-1 扇区编码法，本质是把三个正负判断拼成一个 3bit 编码。三相 SVPWM 里有效组合对应 1~6 六个扇区，`0` 和 `7` 一般当作异常或零矢量情况处理。

确定扇区后，就根据 `sector` 进入对应分支，计算这个扇区内两个相邻有效矢量的作用时间：

```text
T1：第一个有效矢量作用时间
T2：第二个有效矢量作用时间
T0：零矢量时间，T0 = PWM周期 - T1 - T2
```

最后把 `T0` 分到周期两边，再叠加 `T1/T2`，得到三相占空比 `Ta/Tb/Tc`，写进 HRTIM/高级定时器比较寄存器，输出三相互补 PWM。

## 12.1 项目版本

> 我在项目里做的是一个基于 `STM32G4` 的有感 FOC 控制器。硬件侧重点有两个：一是利用 `HRTIM` 产生高频互补 PWM，并配合死区和同步触发；二是利用 `ADC + DMA` 做相电流同步采样，尽量减少采样抖动和 CPU 开销。控制算法链路上，我实现了 `Clarke/Park` 变换、`dq` 电流闭环 PI，以及 `SVPWM` 调制。位置反馈采用编码器角度，先从机械角换算到电角度，再参与 Park 变换。整个系统最终实现了 `M3508` 无刷电机的闭环驱动控制。

## 12.2 如果面试官追问“你具体做了什么”

> 我的工作不只是把公式搬上去，而是把控制链路真正跑起来，包括 PWM 和 ADC 的时序配合、两相电流采样、角度换算、闭环调试，以及让电机从能转到稳定闭环。实际调试时我会重点看角度是否对齐、电流采样是否稳定、电流环有没有振荡，以及占空比输出和实际转速响应是否一致。
