
# 1 C/C++ 与内存

## 1.1 指针与关键字

1. 指针数组与数组指针的定义及区别，分别如何使用？
	指针数组：是数组，里面装的是多个指针，int * p[4]，可以存多个字符串。
	数组指针：是指针，指向一个数组，int (* p)[4]，主要用于二维数组。
	
2. 嵌入式中避免野指针的方法。
	分配指针的时候需要初始化地址，指向NULL；
	free完把指针指向NULL；
	不返回局部变量的地址；
	指针的作用域尽量小，谁用就谁释放。

3. `const` 关键字的用法。
	修饰变量，让变量不可变，变成只读
	在嵌入式中，把变量放去flash，节省RAM
	函数调用的时候，保护输入形参
	
4. `const` 修饰指针时，“指针常量”与“常量指针”的本质差异是什么？
	指针常量是一个常量，const修饰变量int：const * int p ，指针指向一个固定的地址。地址没法变，数值可以变。
	常量指针是一个指针，const修饰指针：const int  * p，指向常量的指针。数值没法变，地址指向可以变。

## 1.2 内存与数据布局

1. C 语言程序运行时的内存分配。
	.text段 CODE，存放代码，在flash
	.data段 RW_DATA，初始有值的变量，存放在flash，reset的时候copy到ram
	.rodata段 RO_DATA，只读的变量（const），存放在flash
	.bss段  ZI_DATA，未初始化或者初始化为0的变量，在RAM
	堆，malloc的，从下往上生长，在RAM，
	栈，存放临时变量从上往下生长，在RAM。
	在RAM里面，
	裸机：从0x20000000开始，是.data，然后.bss，然后堆，栈，然后到栈顶。堆和栈区间是一个缓冲区
	RTOS：使用RTOS的时候，会分配一个静态数组，叫做ucheap，作为RTOS的堆，后续创建任务会用pvPortMalloc在ucheap内部分配一个区域，任务栈和TCB都存储在这里面。这个ucheap在.bss段。
	
2. 堆和栈有什么区别？堆区和栈区之间还有什么区？
	堆，malloc的，从下往上生长，在RAM，
	栈，存放临时变量从上往下生长，在RAM。
	中间还有一个缓冲区。
	
3. 内存泄漏的原因。
	调用了malloc之后没有free；
	指针被覆盖，丢失原有地址

4. 未初始化的全局变量放在哪里？BSS 段省的是什么空间？
	存在bss段里面，省了flash，flash里面不需要存那么多0，只需要统一赋0；
	
5. 结构体对齐：`u8 u16 u32` 和 `u32 u16 u8` 分别占用多少字节？
	 1 + 空 + 2  + 4
	 4 + 2 + 1 + 空
	 都是8字节

6. 如果 MCU RAM 不够了，你会如何分析内存占用？全局变量、堆、栈、RTOS 任务栈分别怎么排查？
	看.map文件，map文件可以看到每个函数和变量占用的RAM，找.bss和.data的数组大小，从大到小排查。
	通过FreeRTOS里面的API获取堆和栈水位。
	解决方法：
	改用联合体，const修饰，采用静态分配

# 2 编译、启动、架构与调试

## 2.1 编译与启动

1. 程序从写完到运行经历什么阶段？
	预处理：展开宏和头文件，取消注释，变成.i
	编译：然后把.i 编译变成汇编码.s
	汇编：生成机器码目标文件.o
	链接：合并多个.o目标文件，变成可执行文件.axf/.elf
	单片机上电，设置MSP，再从下一个四字节取Reset_Handler地址进行复位，再Reset_Handler里，把.data从flash拷贝到RAM，清零.bss段，然后SystemInit( )里面配置时钟，设置VTOR，最后跳转到main.c

2. STM32 的启动流程？
	首先上电后，程序从自举区，地址0x0000000开始。通过boot引脚配置，硬件会映射到不同的为止。boot = 0 x的时候是flash，boot = 0 1的时候是内置bootloader启动，用来isp升级的，boot = 1 1的时候是sram启动。
	以flash为例，上电后自举区映射到0x08000000，设置MSP，再从下一个四字节取Reset_Handler地址进行复位，再Reset_Handler里，把.data从flash拷贝到RAM，清零.bss段，然后SystemInit( )里面配置时钟，设置VTOR，最后跳转到main.c

3. 单片机运行时，Flash 中的什么数据要搬到 RAM 中？
	有初值的数据，即.data数据，搬运到RAM的.data段；
	如果有要求高速运行的RAM函数，也要搬运；
	
4. 如果系统启动时间从 800 ms 优化到 300 ms，你会从 Bootloader、时钟初始化、外设初始化、任务创建、日志输出哪些地方入手？
	先测量：上电很早的地方开始记录，用dwt或者翻转gpio，用逻辑分析仪进行分析，看看哪一段时间有多久。
	1. 一般来说，bootloader可能有延时等待，可以改成通过按键进行触发，默认直接上电。
	2. 各种外设初始化有haldelay的初始化，可以改成上电统一拉起，统一一个时间延迟后再处理，这样并行处理更快
	3. 上电后只创建关键任务，非关键任务后续再创建
## 2.2 MCU 与 ARM

1. ARM 架构：Cortex-M3、ARMv7 等相关内容。
	Cortex-M3和M4都是ARMv7架构，哈佛结构（取指和取数一起）。
	无 MMU（可选 MPU），带 **NVIC** 中断控制器，支持位带操作。
	两种工作模式：Handler（异常）/ Thread（线程）；分特权级和非特权级。
	寄存器，R0-R3，R12为通用寄存器，R13为SP，R14为LR，R15为PC，xPSC。
	
2. 简单说一下用过的 STM32：主频、Flash、RAM。
	STM32F411CEU6，主频100MHZ，Flash 1M，RAM128Kb。
	GD32E225C8t6，主频72MHZ，Flash是64KB，RAM是8Kb
	STM32F407VET6，主频168MHZ，Flash是512KB，RAM是192KB（128KB SRAM + 64KB CCM）
	STM32F407VGT6，主频168MHZ，Flash是1MHZ，RAM是192KB（128KB SRAM + 64KB CCM）
	
3. STM32 中断控制器是什么？
	NVIC，嵌套向量中断控制器，Nested Vectored Interrupt Controller
	支持中断嵌套；
	优先级分组：抢占优先级 + 子优先级（4 位共 16 级可配，数值越小优先级越高）。
	通过中断向量表找 ISR 入口，进入异常时硬件自动压栈 8 个寄存器。
	外部 GPIO 中断先经过 EXTI 控制器再到 NVIC。
## 2.3 调试与故障定位

1. 用过 GDB 吗？常用命令有哪些？
	在VScode里面用过，通过GDB去控制OpenOCD进行板子的调试。
	b断点，r运行，c继续，finish跑完当前函数
	p 打印变量
	
2. 对寄存器的了解，如何查看运行过程中的调用栈？（打断点调试）
	打断点，看register可以看到各寄存器，IDE 的 Call Stack 。

3. 不用调试器怎么判断问题？
	用串口日志。
	通过LED，编码器，GPIO配合逻辑分析仪。
	故障的时候，寄存器的和栈存入noinit RAM或者Flash里面

4. 访问内存出错、Bus Error 这类问题如何排查？
	首先，得看是哪个内核
	Cortex-M4、M7、M3可以用CFSR，内部包含16位的UFSR、8位的MMFSR、8位的BFSR，根据这几个标志位可以查表得到是什么错误类型。
		UF（用法错误），通过栈回溯来看。
		MMF（内存出错），通过MMFAR可以看到出错地址。
		BF（总线错误），通过BFAR可以看到出错地址。
		这些寄存器直接在Keil里面可以看
	 如果是Cortex-M23、Cortex-M0，只能使用栈回溯。
		栈回溯：进入hardfault异常的时候，硬件会把进入异常的时候的寄存器压栈中，此时LR数值是EXC_RETURN。通过查看hardfault的时候的LR寄存器的bit2位是0还是1定位EXC_RETURN在MSP还是PSP里面。如果是0的话，LR最低位为9，此时MSP，如果是1，LR的十六进制末尾是D，此时PSP。通过SP的地址，可以找到栈帧。里面硬件压入r0-r3，r12，lr，pc，xpsr寄存器，共8个，每个4字节。找到PC寄存器，就可以找到进入异常之前的程序地址。在keil反汇编窗口输入这个地址或者去map文件，gdb addr2line进行查找。
	
5. 系统运行几个小时后偶发卡死，但现场无法接调试器，你会在固件里提前设计哪些日志、异常记录和自恢复机制？
	RTOS层面上，我会在所有任务统一发送心跳包给一个健康监测任务，这个任务优先级最高，用来接收各个任务心跳、记录任务栈最高水位、记录最大CPU占用率、堆余量等，然后统一投喂看门狗。
	除此以外还可以看复位原因寄存器（RCC->CSR）。
	如果是系统异常，比如hardfault，那我会把进入hardfault的时候的栈帧给提取出来，比如当前是psp还是msp，然后把r0-r3，r12，lr，pc，xpsr都存flash里或者noinit区的sram里面。后续进行分析

# 3 外设、总线与通信协议

## 3.1 I²C

1. I²C 多主仲裁。
	非同时发起START：
		那就是先发起START的设备优先，另外一个不会对总线进行操作。
	同时发起START：
		IIC是开漏的，SDA发送的时候，会检测总线电平，如果两个设备同时发送，由于开漏的原因，先发0的设备会覆盖掉1的，如果发现被覆盖了就输掉仲裁，先发0的赢得仲裁。所以说，一般写的优先级会高于读的。
		SCL也是开漏的，所以说低电平时间取决于各主机中最长者
	
3. I²C 的通信时序。
	SCL和SDA默认都是最高电平，SCL是高电平有效。在SCL高电平期间，主机把SDA从高到低，为一个开始信号，然后主机开始拉低SCL，在SCL低电平的时候变化SDA，在SCL高电平的时从机进行采样。在主机发送完8位数据之后的下一位，主机会把SDA释放，然后下一个SCL高电平的时候，从机需要把SDA拉下，代表ACK应答。主机可以继续发送。最后结束信号是SCL高电平期间，SDA从低到高。
		
4. I²C 的死锁是啥，怎么恢复？
	IIC运行过程中，由于主机掉电重启、外部干扰、上升速度，导致主机发送完之后，从机刚好把SDA拉低到0等待SCL，状态机卡死在某个位置导致整个总线死锁。
	关闭IIC外设，把两根引脚配置为开漏的GPIO，主机发送SCL，推动从机状态机移动，直到SDA为高的时候，发送STOP信号，结束从机。工程上一般用9个单位的SCL（8位数据+1个ACK），最后重新初始化IIC外设，再重新发起通信。
## 3.2 SPI

1. 讲一讲 SPI 通信中的 CPOL和 CPHA。
	C是clock（时钟）、POL是polarity（极性）、PHA是phase（相位）
	CPOL是时钟极性，CPHA是时钟相位。
	CPOL决定SCK空闲状态时候总线电平高低，0为低电平空闲，1是高电平空闲。
	CPHA决定在空闲之后的什么时候进行采样和什么时候切换数据，0是第一个边沿采样，第二个边沿切换数据，1是第二个边沿采样，第一个边沿切换数据。
	
2.  SPI有几个模式，你用的设备是什么模式。
	CPOL到CPHA来进行排列，00为模式1，11为模式3。
	 bmi088支持mode0和mode3，即00和11。00 clk空闲为0，上升沿采样，下降沿切换数据。11是空闲为1，上升沿采样，下降沿切换数据。对bmi088来说只关心上下边沿。
	 	
3. SPI 有几根线？通信过程是什么？
	4根线，MOSI（主发从收），MISO（主收从发），CLK（时钟线），CS（片选）
	主机选中片选，然后发送CLK，根据CPOL和CPHA的配置，进行数据切换、从机采样。
	主机移位寄存器 - MOSI - 从机移位寄存器 MISO，形成一个回环，通过CLK进行驱动，每个CLK把1bit数据移动。
	以BMI088为例，BMI088的寄存器地址是7位的，第一字节的首位是0/1，0是写，1是读。数据是16bit的。片选选择陀螺仪，发送1+角速度寄存器地址，然后读取48bit，也就是6字节，在这个过程中，主机一直发Dummy推动回环，从机把XYZ的三轴角速度发送过来。下一次通信的时候重复这个步骤。加速度计由于设计原因，需要等待一个空闲dummy，发送1+加速度寄存器地址，然后发送8bit的dummy，丢弃之后，下48bit就是加速度的数据了，连续读。

4. 如果 SPI 接口连接 IMU，采样频率很高，你会如何设计 DMA、缓存、时间戳和数据完整性检查？
	我们采用的BMI088陀螺仪最高2000hz，加速度是1600hz，配置INT引脚为REDY（数据就绪），连接单片机gpio，触发EXTI外部中断，在中断里面释放通知量，来触发陀螺仪和加速度计的采样。
	采样的时候，使用DMA+SPI，任务挂起，阻塞等待信号量。在DMA搬运结束的中断里面释放通知量，再回来处理。
	数据可以采用双缓冲区，每次DMA写完之后通过翻页切换读写角色。DMA在写a缓冲区的时候，IMU数据从b区读取，在dma写b区的时候，IMU数据从a区读取，避免数据写一半被读取造成数据不完整。

## 3.3 UART、CAN 与协议设计

1. 项目中的串口是怎么使用的？
	用的DMA空闲中断接收不定长的数据，配合循环缓冲区。数据接收完了之后，发送通知，任务进行解析。

2. 串口协议栈怎么设计？怎么量化性能指标？
	我一般用到的就是帧头、数据长度、数据、CRC、帧尾。
	看丢帧率、CRC校验失败率。

3. CAN 总线及其优势。
	可以多主机，采用差分信号线，抗干扰强。速度快，能达到1Mhz。FDCAN可以更快。基于ID的自动仲裁机制，ID越小优先级越高（类似于IIC的线与）。包含硬件CRC+ACK。

4. UART/CAN/USB 通信出现偶发丢包，你会如何区分是物理层问题、驱动问题、协议设计问题还是上层处理不及时？
	物理层：示波器看波形质量、幅值、干扰、波特率偏差。还可以看UART的错误寄存器。
	驱动层：看DR，RXE，TXE这些寄存器，看看是否有进入中断，查看是不是数据进了MCU但是驱动问题。
	协议层：有无校验和帧边界错位后能否恢复。
	上层：接收缓冲/队列是否溢出；临时加大缓冲或提高任务优先级做对照实验，丢包消失即上层问题。

5. 让你设计一个上位机和嵌入式设备之间的二进制协议，你会如何考虑帧格式、校验、序号、重传、兼容性和调试便利性？

# 4 中断与 DMA

1. 中断如何产生？中断响应的基本过程是什么？
	由外设产生：gpio边沿、定时器溢出，串口数据。置位中断标志位。中断已使能的话，向NVIC发送请求。
	NVIC决定是否响应，如果响应了，进入异常之前，压入r0-r3，r12，lr，pc，xpsr寄存器。然后从中断向量表里面查找对应的ISR，把PC指向那个地址进行跳转。ISR内部清除标志位，执行完中断之后恢复现场。

2. 外部中断的触发方式有哪些？中断优先级配置需遵循什么原则？
	上边沿、下边沿、上下边沿。
	紧急的优先级更高。
	如果要使用FreeRTOS的API，优先级必须得大于RTOS设定，默认是5。

3. 中断管理如何设计？
	ISR 只做最小工作：清标志、取数据、发通知（信号量/任务通知），耗时处理交给任务。
	全局优先级规划表，统一分配、文档化，避免各模块随意配。
	临界区关中断时间尽量短；

4. 中断服务函数与任务间通信的安全方式是什么？为何不建议在 ISR 中直接调用复杂函数？
	使用FormISR的API，这类API通常是快进快出，不带有阻塞等待功能。并且可以带有环形高优先任务的检测。
	因为ISR内调用复杂函数的话，会阻塞其他同优先级或者更低优先级的中断，同时阻塞FreeRTOS的任务，造成实时性下降。如果时间过长，这次中断没处理完，就又触发中断，会造成bUG。同时部分函数是不可重入的，还会占用MSP栈，不可控。

5. 中断里面可以用 FreeRTOS 函数吗？（一般优先级数值小于 5 就不可以）
	得看BASEPRI，一般是5，5以下的任务RTOS无法控制，5以上的中断归RTOS管理

6. 为什么优先级数值小于 5 不可以？
	FreeRTOS用BASEPRI来保护内核临界区。但是BASEPRI只能屏蔽数值大于阈值的中断。

7. DMA 全满中断和串口空闲中断，当二者都满足触发条件时，会同时将标志位置位然后触发吗？
	不会，会先触发DMA全满，然后在空闲中断，空闲中断需要等待一帧才会触发。而且会根据优先级进行排布

8. 满足 DMA 和空闲中断的触发条件后，如果把中断关闭 10 个字节的时间再打开，或者被更高优先级中断阻塞，两个标志位都还在，这时该怎么处理？
	两个优先级都会被挂起，然后会根据NVIC的优先级进行判断，如果优先级一样就看子优先级，子优先级一样通常就看中断向量表谁更靠前。

# 5 RTOS 与操作系统

## 5.1 为什么使用 RTOS

1. 裸机系统与 RTOS 系统的核心差异是什么？有用 RTOS，为什么要用？（可移植性、资源受限下完成并发任务、可扩展性等角度）
	实时性更强，本身就是一个中间件，可以适配很多芯片。可以节省定时器资源。

2. RTOS 是如何保证实时性的？（任务优先级、中断、锁、信号量、任务调度等）
	抢占式调度和时间片轮询。通过消息通知，信号量，队列进行任务的同步。互斥锁进行资源保护，信号量和通知量由事件驱动。同时使用systick保证任务的节拍。

## 5.2 任务状态与调度

1. RTOS 中任务的状态有哪些？
	运行、就绪、挂起、阻塞

2. RTOS 任务运行机制是什么？RTOS 如何保证任务调度？（任务调度器、优先级、时间片轮转、上下文切换、阻塞唤醒等）
	每个任务都有自己的任务控制块TCB，里面有任务的信息。同时每个任务有自己的任务栈。通过systick提供节拍，并通过PendSV异常进行上下文切换。使用抢占式调度和时间片轮转进行任务切换。
	任务切换可以通过时间片、任务延时到、信号量通知量队列等进行触发。

3. 高优先级任务是如何抢占低优先级任务的？
	当一个高优先级被比如systick、信号量、通知量、任务延时的唤醒了，加入去了就绪列表。在调度点，调度器看此时任务的 优先级如果小于就绪列表里面的最高优先任务，就挂起PendSV，进行任务切换

4. 任务优先级的分配原则是什么？高优先级任务长期占用 CPU 会导致什么问题？
	最高优先级给紧急任务，比如保护任务、健康监测任务这种。CPU密集型任务放在低优先级。最低优先级放那些不紧急的任务，比如栈流水记录。
	长期占用CPU会导致别的任务无法保证实时性。导致低优先级任务无法运行。

5. 如果你负责一个飞控或云台控制相关模块，如何保证 1 ms / 5 ms / 10 ms 级周期任务的实时性？
	`vTaskDelayUntil`去作为时间控制。1ms最高，5ms任务次之、10ms最后。
	最高优先级是安全监测系统。其次是陀螺仪和加速度计的读取和IMU融合。保证最主要的数据能够获取。然后飞控自稳。中断只释放信号量、通知量等，解析和控制放在任务里

## 5.3 任务通信与同步

1. RTOS/FreeRTOS 任务之间通信是怎么做的？用过哪些方式、什么场景？（任务通知、消息队列、信号量、互斥锁；平时用什么比较多？）
	在无动力有控制导飞镖里面，用的个比较多。在陀螺仪数据获取的时候用到了通知量、互斥量、信号量、事件组。保证数据获取->控制的时序以及线程安全。

2. 消息队列的发送与接收过程是什么？如何处理队列满/空的异常情况？
	发送就是把数据copy到队列尾部，接收就是从队列头部取出。
	队列满了，可以选择阻塞等待、阻塞超时、直接返回失败。发送方可以强制覆盖。
	空的情况下直接阻塞等待或则设置阻塞超时就行了。

3. 信号量的两种类型分别适用于什么场景？
	一个是信号量一个是互斥量，根据pcHead进行区分，信号量主要分为二值信号量和计数信号量，用于任务同步与计数。互斥量带有优先级继承，资源保护。

4. 信号量和互斥锁有什么区别？底层是怎么设计的？
	信号量和互斥量都是一种特殊的队列，长度为1，size为0的队列，
	信号量通过uxMessagesWaiting进行计数，二值信号量的上限为1，计数信号量的上限为n（计数上限）。
	互斥量创建的时候，队列的pcHead = NULL，来区分为互斥量。如果是互斥量，队列的结构体里面的结构体将会从环形缓冲区变成互斥量被谁拿了，和递归互斥量数量。在调用互斥量，如果调用失败，会去看互斥量现在被谁持有，如果那个任务比自己优先级更低，就会短暂的提升那个任务的优先级，让那个任务释放互斥量，然后恢复任务优先级。完成优先级继承。

5. 多任务/多线程运行时需要注意什么？线程同步用过什么？资源竞争如何通过同步和互斥保证安全？
	需要注意数据竞争，死锁，优先级反转等等。

6. 临界区数据处理：互斥锁与条件判断如何选择？
	临界区开销很小，但是会关BASEPRI阈值以上的中断，阻塞其他任务的执行，建议是短时间并且需要保护的操作。比如CAN发送。
	互斥锁不影响其他任务的运行，不影响中断响应。

7. 一个控制任务和通信任务同时访问同一份状态数据，如何设计数据同步？什么时候用互斥锁，什么时候用消息队列或双缓冲？
	如果是是只读的话，没所谓。
	互斥锁:  加锁保护共享数据(通用但读写互等)
	双缓冲:  写一块读另一块(要最新快照、无锁不阻塞)
	环形缓冲: 数据排队不丢(连续流、单读单写可无锁)

8. 自旋锁和互斥锁有什么区别？
	互斥锁等待的时候是阻塞等待，释放CPU。
	自旋锁让CPU空转等待
## 5.4 上下文切换

1. 了解任务上下文切换的具体过程吗？上下文切换时系统做了什么？线程切换是怎么设计的？
	了解。systick、信号量、队列、通知量、任务延时唤醒作为调度时机，挂起PendSV中断，在PendSV中断触发的时候，会把此时的调用者寄存器压入当前的任务栈中，进入异常，然后切换成MSP。在PendSV异常里面，把r4-r11软件存储去当前任务的PSP栈中，并且更新当前任务TCB里面的PSP栈顶指针。然后把current TCB指向改成就绪列表里面的下一个任务的TCB，从下一个任务的PSP栈里面恢复r4-r11，更新PSP栈指针，然后退出异常。退出的时候硬件自动恢复r0-r3，r12，lr，pc，xPSR。完成上下文切换。

2. 上下文切换源码中有汇编代码，了解哪些相关汇编指令？
	b是跳转函数，bl是跳转函数之前存储lr值，bx是跳转寄存器内地址
	mrs是把寄存器写进psp
	msr是把psp写进寄存器
