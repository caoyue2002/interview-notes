# RX101 AT32F403ACGT7 — FreeRTOS 任务说明文档

> 基于源码直接整理，不依赖注释。所有优先级、栈大小、周期均来自 `freertos_app.c` 的 `xTaskCreate` 调用。

---

## 任务一览

| 任务名            | 函数                 | 优先级 | 栈(words) | 周期         |

## 1. my_task01（优先级 0）

**做什么：** 心跳信号输出。

初始化 PA6（AP2PM_HB_3V3）为推挽输出并置高，然后每 500ms 翻转一次电平，输出 1Hz 方波。无其他逻辑。

**代码依据：** `freertos_app.c` 的 `my_task01_func`，`gpio_bits_toggle(GPIOA, GPIO_PINS_6)` + `vTaskDelay(pdMS_TO_TICKS(500))`。

## 2. RX_moto_ctrl（优先级 7，1kHz）

**做什么：** 向 29 个电机发送 MIT 控制帧，接收电机状态反馈，是实时性要求最高的任务。

**主循环流程（每 1ms，由 `vTaskDelayUntil` 保证绝对时间节拍）：**

### 阶段 0：读取上位机目标值

从 `host_comm_task` 写入的共享静态数组 `target_pos / target_vel / target_kp / target_kd / target_tau[29]` 读取本周期控制目标（float，弧度/弧度每秒）。

### 阶段 1：检查上一周期 RX 窗口

遍历 6 个端口，检查上一周期发出 MIT 帧后的回复情况：
- 全部收到（`rx_received >= rx_expected`）：正常关窗。
- 超过 `RX_DEADLINE_US` 未收齐：异常关窗，标记超时电机，记录丢包计数，本端口本周期**跳过发送**。
- 未到 deadline 也未收齐：继续等待，本端口**跳过发送**。

### 阶段 2：构建 MIT 控制帧

遍历 6 个端口，对每个端口的电机：
- 用 `RAD_TO_ESC_POS(rad)` 将 float 弧度转为 ESC int32 count（LSB = 2π/65536）。
- 用 `RADS_TO_ESC_VEL(rads)` 将 float 弧度/s 转为 ESC int16 count。
- 区分 F 电机（hip pitch/knee）和 S 电机（其余），使用不同的 `CKP/CKD` 增益系数将 host kp/kd 换算为 ESC int16 count。
- **脚踝电机覆盖**（flat 16/17/22/23）：pos=0, vel=0, kp=0，tau 由 `parallel_pd_task` 的结果（经 `parallel_pd_task_get_result_foot`）提供，kd 保留 host 值。
- **腰部并联电机覆盖**（flat 24/25）：同上，tau 来自 `parallel_pd_task_get_result_waist`。
- 覆盖后调 `MDV0V1_BuildV0Frame` 打包成 V0 帧写入该端口的 `tx_buf`。

### 阶段 3：按序发送 MIT 帧

按 PORT_1→2→3→4→5→6 顺序调 `uart_comm_send_tx_buf`，通过 TDC 中断 + 信号量确认发完后 RS485 切回 RX 模式，记录发送时间戳，打开 RX 接收窗口（设置 `rx_expected`）。

### 阶段 4：电机状态接收（异步，IDLE ISR 写入）

ESC 收到 MIT 帧后约 10µs 开始串行回复，IDLE 中断驱动 `uart_comm_idle_irq_handler` 解析帧，通过注册的 V0 handler `process_status_frame` 将数据写入全局数组 `g_port_motor_status[port][motor_id]`，同时更新 `s_motor_rx_us` 时间戳和 `s_port_motor_replied` 位掩码。

### 数据保护

`g_port_motor_status` 由 ISR 写、任务读，float 在 Cortex-M4 是单周期写，ISR 和任务不并发（ISR 在任务主循环之间的间隙执行），实际无撕裂风险。并联电机 tau 快照用 `taskENTER_CRITICAL` 保护。

## 3. RX_parallel_pd（优先级 4，1kHz）

**做什么：** 为脚踝和腰部并联机构执行正运动学（FK）+ PD 控制计算，输出力矩供 moto_ctrl 使用。

**主循环流程（每 1ms，由 `xTaskDelayUntil` 保证）：**

1. 用 `taskENTER_CRITICAL` 快照当前输入：左脚/右脚/腰部各自的 `motor_pos[2], motor_vel[2], joint_target[2], kp[2]`。
2. 串行计算：`parallel_ankle_pd`（右脚）→ `parallel_ankle_pd`（左脚）→ `parallel_waist_pd`（腰部）。纯数学计算，FK LUT 查表 + Jacobian，无任何阻塞调用和外设访问。
3. 用 `taskENTER_CRITICAL` 写回结果（`ParallelPDResult`），推进全局 `s_publish_seq`，三个结果共用同一 seq。
4. 更新统计：`last_cost_us, avg_cost_us, max_cost_us, overrun_count（>2ms）, period_miss_count`。

**输入来源：** `moto_ctrl_task` 在每周期构建 MIT 帧时，从 `g_port_motor_status` 读取电机反馈的 angle/vel，换算成 rad 后调 `parallel_pd_task_set_input_foot/waist` 写入（同样在 `taskENTER_CRITICAL` 内）。

**输出去向：** `moto_ctrl_task` 构建 MIT 帧时调 `parallel_pd_task_get_result_foot/waist` 读取 `tau_motor`，覆盖到脚踝/腰部电机的 MIT 帧。

## 4. RX_host_comm（优先级 5，默认 5ms/200Hz）

**做什么：** 与上位机之间的全双工通信——接收上位机命令、下发控制目标，采集状态数据、上报 feedback。

**主循环流程（由 `vTaskDelayUntil` 保证周期，周期 `s_tx_period_ms` 默认 5ms）：**

### RX：轮询解析上位机命令

调 `host_comm_process_rx`：计算 `DMA1_CH1->dtcnt` 得到 DMA ring buffer（4096B，循环模式）的 head 位置，与 `s_rx_dma_tail` 做差，将新增数据 memcpy 到线性缓冲区，扫描 SOF（0x55）+ 读取帧长度，找到完整 V1 帧后调 `MDV0V1_Process` 分发到对应 CMD handler。

关键 CMD handler：
- **CMD#0903**（`host_comm_cmd_target_pos`）：解压 int16→float，写入 `target_pos/vel/kp/kd/tau[29]` 共享数组。
- 其他命令（enable/disable feedback、设零点、参数调整、IMU 标定、DFU 等）在各自 handler 中处理。

应答帧不在 handler 回调里直接触发 DMA，而是 memcpy 到 `s_resp_tx_buf` 并置 `s_resp_pending` 标志，等主循环安全点发送。

### TX：发送延迟应答

若 `s_resp_pending` 为真，调 `host_comm_dma_send` 通过 DMA1_CH2 发出应答帧，通过 `s_dma_done_sem` 信号量（DMA TC ISR 释放）等待完成。

### TX：发送 feedback 帧

调 `host_comm_collect_feedback` 采集：从 `g_port_motor_status[6][6]` 读取所有电机当前状态，从 IMU 共享结构体读取姿态角，打包成 `md_humanoid_mcu_feedback_rsp_t`。从 LOG 队列取一条日志粘连在 feedback 帧尾（`log_data + log_len`）。

用 ping-pong 双缓冲 `s_tx_buf[2]`：调 `MDV0V1_BuildV1Frame` 打 V1 帧头+CRC 写入 `s_tx_buf[s_tx_fill_idx]`，发送后翻转 `s_tx_fill_idx`，DMA 发旧帧时 CPU 已可填下一帧。

**UART7 TX DMA 共享：** `g_host_comm_enable=1` 时本任务使用 DMA1_CH2，`debug_task` 跳过发送；`g_host_comm_enable=0` 时反之。

## 5. RX_imu（优先级 3，事件驱动）

**做什么：** 采集 ICM42670 IMU 数据，经滤波和姿态融合后输出供上层使用。

**主循环流程（阻塞在 DRDY 信号量上）：**

1. 调 `imu_int_wait_drdy(10ms timeout)` 阻塞等待二值信号量。
2. PA8 EXTI 上升沿中断（ICM42670 DRDY 引脚）触发 `imu_int_drdy_isr`，`xSemaphoreGiveFromISR` + `portYIELD_FROM_ISR` 唤醒任务。
3. 任务唤醒后调 `imu_app_process`：
   - `imu_driver`（SPI2 CPU 轮询，12 字节，每寄存器独立 CS 事务）读原始数据。
   - `imu_base`：int16 raw → 物理量（m/s²，dps），附 TMR1 64-bit 微秒时间戳。
   - `imu_axis_remap`：坐标轴重映射。
   - `imu_app`：LPF/MA 滤波。
   - `imu_ahrs`（FusionAhrs）：四元数姿态融合，输出 roll/pitch/yaw + 线性加速度。
4. 结果通过 `imu_app_get_data` 存入共享结构体，`host_comm_task` 和 `sys_stats_task` 读取。

**开机标定流程：** 任务启动时检查 Flash 是否有有效标定数据；若无，挂起 `RX_moto_ctrl / DEBUG / RX_sys_stats` 三个任务，执行六面在线标定（持续喂 IMU 数据直到 `CALIB_STATE_RECORDING` 完成），写入 Flash，恢复被挂起的任务。

## 6. RX_sys_stats（优先级 1，500ms）

**做什么：** 系统监控，定期采集健康状态并上报事件给上位机。

**主循环流程（每 500ms，`vTaskDelay`）：**

1. **计算 CPU 占用率：** 用 `ulTaskGetIdleRunTimeCounter()` 读 IDLE 任务的 runtime counter，与 `portGET_RUN_TIME_COUNTER_VALUE()` 读总 runtime 做差分，计算 `(1 - delta_idle/delta_total) × 100`。不调用 `uxTaskGetSystemState`（会 suspend 调度器 ~300µs 阻塞高优先级任务）。
2. **扫描电机故障：** 遍历 `g_port_motor_status[6][6]` 的 `error_code`，非零则置位故障掩码。
3. **扫描电机温度：** 遍历 `board_temp / motor_temp`，取最大值。
4. **检查 IMU 状态：** 调 `imu_task_get_data` 检查返回值。
5. **检查急停：** 调 `estop_is_triggered()`。
6. **按优先级构建事件帧：** ESTOP > 电机故障 > IMU 故障 > 温度过高（驱动板 80°C / 电机 100°C）> CPU 过高（80%）> INFO 心跳。
7. **发送限频：** INFO 事件每 1s 最多发一次；IMU 故障有 5s 心跳机制，故障期间每 5s 上报一次，中间周期不降级发温度/CPU 事件（防止上位机误判恢复）。
8. 调 `host_comm_send_sys_event`，实际通过 `md_log_tag_urgent` 进入 LOG error 队列队首，由 host_comm 的 feedback 帧粘连带出。

---

## 7. DEBUG（优先级 1，100Hz）

**做什么：** 调试专用，向 UART7 输出 VOFA+ JustFloat 格式波形数据，供上位机调试工具可视化。

**主循环流程（每 10ms，`vTaskDelay`）：**

- 若 `g_host_comm_enable == 1`：直接 `vTaskDelay(10ms)` 跳过，不抢占 DMA1_CH2。
- 若 `g_host_comm_enable == 0`：根据 `g_debug_wave` 变量选择波形模式，打包 float 数组 + VOFA+ 帧尾（`0x00 0x00 0x80 0x7F`），通过 DMA1_CH2 ping-pong 双缓冲发送。

**波形模式（`g_debug_wave`）：**
- 0：IMU 加速度 ax/ay/az（m/s²）
- 1：选定端口（`g_debug_port`）的每电机丢包率
- 2：IMU 陀螺仪 gx/gy/gz（dps）
- 3：最后更新的电机 angle/vel/real_tor
- 4：IMU 原始 ADC 值 + 采样耗时（µs）
- 5：IMU 全量（加速度 + 陀螺仪）
- 6：ESTOP 标志 + PA15 电平
- 7：PC14/PA1 复位信号电平
- 8：硬件版本号 + ADC 原始值
- 9：PA6 心跳电平
- 12：IMU 四元数 w/x/y/z
- 13：并联 PD 结果（joint_pos/vel, tau_motor）+ 任务耗时统计

## 任务间数据流汇总

```
上位机（UART7 4Mbaud）
  │ CMD#0903 V1帧
  ▼
host_comm_task（优先级5，5ms）
  │ target_pos/vel/kp/kd/tau[29]（共享数组）
  ▼
moto_ctrl_task（优先级7，1ms）
  │ MIT控制帧（V0协议）
  ▼
RS485 六路总线（6Mbaud）──→ 29个ESC电机
  │ 状态反馈帧（IDLE ISR异步）
  ▼
g_port_motor_status[6][6]（共享数组）
  │
  ├──→ host_comm_task：打包feedback上报上位机
  ├──→ moto_ctrl_task：读取反馈驱动parallel_pd输入
  └──→ sys_stats_task：扫描故障/温度

parallel_pd_task（优先级4，1ms）
  │ 输入：g_port_motor_status → motor_pos/vel（由moto_ctrl写入）
  │ 输出：ParallelPDResult（tau_motor）
  ▼
moto_ctrl_task：覆盖脚踝/腰部MIT帧tau字段

imu_task（优先级3，DRDY驱动）
  │ SPI2读取 → 滤波 → FusionAhrs → imu_data_t
  ▼
  ├──→ host_comm_task：打包进feedback帧
  └──→ sys_stats_task：检查IMU健康状态
```
