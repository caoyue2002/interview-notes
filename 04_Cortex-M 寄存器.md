# 1 寄存器总览

```
R0 ~ R12   通用寄存器，用于中间值计算
R13 (SP)   堆栈指针，内部分MSP/PSP两个独立寄存器
R14 (LR)   链接寄存器，保存函数返回地址
R15 (PC)   程序计数器，存放下一条指令地址
```
在STM32单片机下，全部都是 **32位**

---
# 2 R0-R12 通用寄存器

用于中间值计算，编译器调度使用，没有固定含义（不像SP/LR/PC有专属职责）。
ARM调用约定（AAPCS）里有几个常用角色：

- **R0**：函数第一个参数 / 返回值
- **R0-R3**：函数调用时的前4个参数
- **R12**：有时作为子程序调用的临时寄存器（IP，Intra-Procedure call scratch register）

---

# 3 SP（R13）堆栈指针寄存器

```
R13 对应两个完全独立的32位物理寄存器：
  1.MSP（主栈指针，Main Stack Pointer）
  2.PSP（进程栈指针，Process Stack Pointer）
```

两者地址（各自存的栈顶值）通常不同；实际操作哪个由 **CONTROL寄存器的SPSEL位**决定，硬件自动路由。

## 3.1 为什么需要两个

> [!important] 
根本原因是**中断嵌套符合LIFO**（后进先出，栈的天然特性：先发生的中断必然最后处理完），可以直接用压栈/弹栈处理；**任务调度不符合LIFO**（可能任意顺序恢复，由优先级/时间片决定），必须靠PSP配合每个任务TCB里的`pxTopOfStack`。所以拆成两个：MSP固定服务异常中断，PSP跟着任务切换。"

|      | MSP          | PSP           |
| ---- | ------------ | ------------- |
| 谁用   | 异常、中断、裸机调用函数 | 各任务运行时        |
| 指向   | 系统栈（固定区域）    | 当前任务的栈（随切换变化） |
| 恢复顺序 | LIFO（栈天然支持）  | 任意顺序（需TCB记账）  |

## 3.2 MSP溢出 vs PSP溢出——风险来源不同

|     | MSP栈溢出             | PSP栈溢出（某个任务）                            |
| --- | ------------------ | --------------------------------------- |
| 原因  | 中断嵌套层数太深、裸机下函数调用太深 | 这个任务自己的函数调用/局部变量用栈太多                    |
| 影响  | 全局性                | 局部性——通常只影响这一个任务，但是越界可能影响到相邻的任务TCB，引发Bug |
| 预防  | 限制中断嵌套深度，给系统栈留够空间  | 每个任务创建时合理估计栈大小（xTaskCreate的栈大小参数）       |
# 4 LR（R14）链接寄存器

- BL fun( )：跳转到`fun()`，同时把`fun()`下一行指令的地址存入LR。
- BX LR：把LR赋给PC。

如果`fun()`自己又调用了别的函数（嵌套调用）：会把当前LR压栈保存——因为马上要执行的BL会覆盖LR 。
被调用的函数返回后，`fun()`再从栈里弹出LR，最后执行BX LR，才能正确跳回。

> [!important] 异常时LR = EXC_RETURN，不是返回地址 进异常时LR被硬件写入特殊值（如`0xFFFFFFF9`/`0xFFFFFFFD`），靠bit2告诉CPU该用MSP还是PSP恢复（9=MSP，D=PSP）。这是Hardfault判断MSP/PSP的核心机制。

---

# 5 PC（R15）程序计数器

存"CPU下一个要取的指令地址"。

**正常执行**：每执行完一条指令，PC自动+2（Thumb指令16位/2字节），顺序往后走。

**跳转指令**（`BL`/`B`/`BX`）：是**直接把PC改写成目标地址**——不管目标离多远，瞬间跳过去，下次CPU就从新地址开始取指。

---

# 6 xPSR状态寄存器

存的是CPU的"状态信息"而不是计算结果本身。APSR里的N/Z/C/V标志位记录上次运算的状态（是否为0、是否负数、是否溢出），如果不保护好，中断处理过程会污染这些标志位，导致返回后条件判断出错。

---

# 7 现场保护机制

## 7.1 函数调用

ARM调用约定（AAPCS）把寄存器分两类：

```
调用者保存(caller-saved): R0-R3, R12
  → 没人天然替你保护，自己想留就自己压栈

被调用者保存(callee-saved): R4-R11
  → 被调用函数如果要用，自己负责压栈保存、用完恢复
```

## 7.2 中断

> [!important] 硬件自动保存8个：R0-R3, R12, LR, PC, xPSR 软件还需额外保存8个：**R4-R11**（异常处理程序自己要用才需要，FreeRTOS在`PendSV`里手动压）

| 寄存器        | 保存方式             | 属于哪类     |
| ---------- | ---------------- | -------- |
| R0-R3, R12 | 硬件自动             | 调用者保存类   |
| LR         | 硬件自动             | 返回记号     |
| PC         | 硬件自动             | 打断位置     |
| xPSR       | 硬件自动             | 打断时的状态标志 |
| R4-R11     | **软件手动**（PendSV） | 被调用者保存类  |
