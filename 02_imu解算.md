# 四元数姿态解算内容总结

---
## 1. 核心目标

姿态解算的目标是：

> 根据 IMU 传感器中的加速度计和陀螺仪数据，实时计算物体当前的姿态角，也就是欧拉角。

常见输出包括：

- **Roll**：横滚角
- **Pitch**：俯仰角
- **Yaw**：偏航角

在姿态解算中，通常不用欧拉角直接积分，而是使用**四元数**来描述姿态，最后再将四元数转换成欧拉角。

---

## 2. 为什么使用四元数

四元数可以表示空间旋转，形式为：

```
q = q₀ + q₁i + q₂j + q₃k
```

也可以写成：

```
q = [q₀, q₁, q₂, q₃]
```

其中：

- **q₀**：实部
- **q₁, q₂, q₃**：虚部

**使用四元数的优点：**

| 优点 | 说明 |
|------|------|
| 避免万向节死锁 | 欧拉角的致命缺陷 |
| 计算量更小 | 比旋转矩阵效率高 |
| 适合嵌入式系统 | 实时姿态解算 |
| 方便积分更新 | 通过陀螺仪角速度直接积分 |

---

## 3. 初始四元数

程序开始时，一般将四元数初始化为：

```
q₀ = 1,  q₁ = 0,  q₂ = 0,  q₃ = 0
```

对应代码为：

```c
q0 = 1.0f;
q1 = 0.0f;
q2 = 0.0f;
q3 = 0.0f;
```

这个四元数表示：**初始姿态没有旋转，机体坐标系和导航坐标系默认对齐。**

---

## 4. 读取 IMU 传感器数据

姿态解算主要使用两类数据：

### 4.1 加速度计数据

```
ax, ay, az
```

表示机体坐标系下三个方向的加速度。

> 在静止或低动态情况下，加速度计主要测量的是**重力加速度方向**。

### 4.2 陀螺仪数据

```
gx, gy, gz
```

表示绕三个轴的角速度。

> 陀螺仪用于积分更新姿态，但长期使用会产生**漂移**。

---

## 5. 陀螺仪角速度单位转换

很多陀螺仪输出单位是 **°/s（度/秒）**，但四元数微分方程使用**弧度制**，需要转换为 **rad/s**。

转换关系：

```
1° = π/180 ≈ 0.01745 rad
```

代码中通常写为：

```c
gx = gx * 0.0174533f;
gy = gy * 0.0174533f;
gz = gz * 0.0174533f;
```

---

## 6. 加速度计归一化

加速度计原始向量为：

```
a = (ax, ay, az)
```

模长为：

```
|a| = √(ax² + ay² + az²)
```

归一化后：

```
ax' = ax / |a|
ay' = ay / |a|
az' = az / |a|
```

代码形式为：

```c
norm = invSqrt(ax * ax + ay * ay + az * az);
ax *= norm;
ay *= norm;
az *= norm;
```

其中 `invSqrt(x)` 表示快速求倒数平方根，即 `1/√x`。

> **加速度归一化的目的：只保留重力方向，不关心重力大小。**

---

## 7. 导航坐标系中的标准重力方向

在导航坐标系中，归一化后的标准重力加速度通常表示为：

```
gⁿ = (0, 0, 1)ᵀ
```

也就是说，**重力方向在导航坐标系中是已知的**（垂直向下）。

---

## 8. 由四元数计算理论重力方向

四元数可以转换成旋转矩阵 C_b^n，它表示机体坐标系和导航坐标系之间的旋转关系。

如果已知导航坐标系中的标准重力方向：

```
gⁿ = (0, 0, 1)ᵀ
```

那么可以通过旋转矩阵的逆，把重力方向转换到机体坐标系下：

```
g_theoryᵇ = (C_b^n)⁻¹ · gⁿ
```

因为旋转矩阵的逆等于转置，所以也可以写成：

```
g_theoryᵇ = (C_b^n)ᵀ · gⁿ
```

> 这个量表示：**根据当前四元数姿态估计出来的理论重力方向。**

---

## 9. 加速度计测得实际重力方向

加速度计归一化后得到：

```
g_actualᵇ = (ax, ay, az)
```

这个量表示：**加速度计实际测得的重力方向。**

| 向量 | 来源 | 含义 |
|------|------|------|
| `g_theoryᵇ` | 四元数计算 | 理论重力方向（我"认为"重力在哪） |
| `g_actualᵇ` | 加速度计 | 实际重力方向（重力"实际"在哪） |

> - 如果当前四元数估计准确 → 两个向量方向一致
> - 如果两者方向不一致 → 说明当前姿态估计存在误差

---

## 10. 姿态误差的构造

两个向量之间的夹角可以表示为：

```
θ = arccos( g_theoryᵇ · g_actualᵇ )
```

| θ 值 | 含义 |
|------|------|
| θ = 0 | 两个向量方向一致，姿态估计正确 |
| θ ≠ 0 | 两个向量存在夹角，姿态估计有误差 |

---

## 11. 为什么用叉乘表示误差

在实际姿态解算中，通常**不用反余弦函数直接求夹角**，而是用两个向量的**叉乘**来构造误差。

叉乘定义为：

```
e = g_actualᵇ × g_theoryᵇ
```

叉乘模长满足：

```
|e| = |g_actualᵇ| · |g_theoryᵇ| · sinθ
```

因为两个向量都已经归一化（模长 = 1）：

```
|e| = sinθ
```

当角度较小时 `sinθ ≈ θ`，因此：

```
|e| ≈ θ
```

> **叉乘可以近似表示两个向量之间的角度误差。**

---

## 12. 叉乘误差的意义

叉乘结果不仅表示**误差大小**，还表示**误差方向**：

| 叉乘输出 | 含义 |
|----------|------|
| 叉乘的**模长** | 姿态误差有多大 |
| 叉乘的**方向** | 应该绕哪个轴修正姿态 |

因此，叉乘误差非常适合用于姿态解算中的**反馈修正**。

代码中常见形式为：

```c
ex = ay * vz - az * vy;
ey = az * vx - ax * vz;
ez = ax * vy - ay * vx;
```

其中：

| 变量 | 含义 |
|------|------|
| `(ax, ay, az)` | 实际测得的重力方向 |
| `(vx, vy, vz)` | 由四元数计算得到的理论重力方向 |
| `(ex, ey, ez)` | 姿态误差 |

---

## 13. 误差积分

姿态误差可以进行积分，用于补偿陀螺仪的长期零偏漂移。类似 PID 控制中的**积分项**。

代码形式为：

```c
exInt += ex * Ki;
eyInt += ey * Ki;
ezInt += ez * Ki;
```

| 变量 | 含义 |
|------|------|
| `Ki` | 积分系数 |
| `exInt, eyInt, ezInt` | 误差积分项 |

> **误差积分的作用：消除陀螺仪长期漂移带来的稳定误差。**

---

## 14. 互补滤波思想

陀螺仪和加速度计各有优缺点。

### 14.1 陀螺仪特点

| 优点 | 缺点 |
|------|------|
| 短时间响应快 | 长时间积分会漂移 |
| 动态性能好 | 存在零偏误差 |
| 适合快速姿态变化 | — |

### 14.2 加速度计特点

| 优点 | 缺点 |
|------|------|
| 静止/低动态时可反映重力方向 | 容易受运动加速度干扰 |
| 长期稳定性好 | 动态过程中不一定只测到重力 |

### 14.3 互补滤波核心公式

因此需要用互补滤波把两者结合起来：

```
ω_corrected = ω_gyro + Kp·e + Ki·∫e
```

也就是：**用加速度计构造出的姿态误差，对陀螺仪角速度进行修正。**

代码形式通常为：

```c
gx = gx + Kp * ex + exInt;
gy = gy + Kp * ey + eyInt;
gz = gz + Kp * ez + ezInt;
```

| 项 | 含义 |
|----|------|
| `Kp * e` | 比例修正项（快速响应） |
| `Ki * ∫e` | 积分修正项（消除稳态误差） |

---

## 15. 四元数微分方程更新姿态

修正后的角速度用于更新四元数。

四元数微分方程可以表示为：

```
q̇ = ½ · q ⊗ ω
```

其中：

| 符号 | 含义 |
|------|------|
| `q` | 当前四元数 |
| `ω` | 陀螺仪角速度构成的纯四元数 |
| `⊗` | 四元数乘法 |

程序中根据采样周期 `dt` 对四元数进行积分更新：

```c
q0 = q0 + qDot0 * dt;
q1 = q1 + qDot1 * dt;
q2 = q2 + qDot2 * dt;
q3 = q3 + qDot3 * dt;
```

> 可以理解为：陀螺仪告诉系统当前正在以什么角速度旋转，程序根据这个角速度不断更新四元数。

---

## 16. 四元数归一化

由于数值积分会产生误差，四元数更新后模长可能不再等于 1。因此每次更新后需要重新归一化：

```
q_normalized = q / √(q₀² + q₁² + q₂² + q₃²)
```

代码形式为：

```c
norm = invSqrt(q0 * q0 + q1 * q1 + q2 * q2 + q3 * q3);
q0 *= norm;
q1 *= norm;
q2 *= norm;
q3 *= norm;
```

> **四元数归一化的目的：保证四元数始终是合法的单位四元数，从而正确表示空间旋转。**

---

## 17. 四元数转换为欧拉角

最后，将四元数转换成欧拉角，得到最终姿态输出：

```
roll  = atan2(2*(q₀*q₁ + q₂*q₃), 1 - 2*(q₁² + q₂²))
pitch = asin(2*(q₀*q₂ - q₃*q₁))
yaw   = atan2(2*(q₀*q₃ + q₁*q₂), 1 - 2*(q₂² + q₃²))
```

如果需要角度制，还要乘以 `180/π`。

---

## 18. 程序整体流程总结

```
1. 初始化四元数：q₀=1, q₁=0, q₂=0, q₃=0

2. 读取 IMU 数据：ax, ay, az（加速度）；gx, gy, gz（角速度）

3. 陀螺仪角速度 角度制 → 弧度制：gx/gz/gz *= 0.0174533

4. 加速度计数据归一化 → 得到实际重力方向

5. 根据当前四元数计算理论重力方向

6. 实际重力方向 × 理论重力方向（叉乘）→ 得到姿态误差 e

7. 对姿态误差进行积分 → 补偿陀螺仪零偏

8. 使用比例积分反馈修正陀螺仪角速度
      g = g + Kp*e + Ki*∫e

9. 利用修正后的角速度更新四元数（四元数微分方程积分）

10. 对四元数进行归一化

11. 将四元数转换成 Roll / Pitch / Yaw 欧拉角输出
```

---

## 19. 思维导图对应关系

**从结果往前推（理解思路）：**

```
欧拉角
  ↓
四元数
  ↓
姿态误差
  ↓
理论重力方向 与 实际重力方向
  ↓
加速度计 和 陀螺仪数据
```

**从输入往输出推（代码流程）：**

```
读取 IMU 数据
  ↓
加速度归一化 → 得到实际重力方向
  ↓
四元数 → 计算理论重力方向
  ↓
理论重力方向 × 实际重力方向（叉乘）
  ↓
得到姿态误差 e
  ↓
误差积分（∫e）
  ↓
互补滤波修正陀螺仪（g = g + Kp·e + Ki·∫e）
  ↓
四元数微分方程积分（q̇ = ½·q⊗ω）
  ↓
四元数归一化
  ↓
四元数转欧拉角 → Roll / Pitch / Yaw
```

---

## 20. 一句话总括

> 四元数姿态解算的核心思想是：**陀螺仪负责短时间内快速更新姿态，加速度计负责提供长期稳定的重力方向参考。** 通过当前四元数计算理论重力方向，再与加速度计测得的实际重力方向做叉乘得到姿态误差，用该误差反馈修正陀螺仪角速度，最后积分更新四元数，并转换为欧拉角输出。

---

## 21. 面试回答版

> 四元数姿态解算主要是利用陀螺仪和加速度计进行互补融合。
>
> 首先读取 IMU 的三轴加速度和三轴角速度数据，将陀螺仪角速度转换为弧度制，并对加速度计数据归一化，得到实际重力方向。
>
> 然后根据当前四元数计算理论重力方向，将理论重力方向和实际重力方向做**叉乘**，得到姿态误差。这个误差通过**比例积分反馈（Kp·e + Ki·∫e）**修正陀螺仪角速度，从而抑制陀螺仪积分漂移。
>
> 之后利用修正后的角速度求解四元数微分方程（q̇ = ½·q⊗ω），更新四元数，并对四元数进行归一化，最后将四元数转换成 Roll、Pitch、Yaw 欧拉角输出。
