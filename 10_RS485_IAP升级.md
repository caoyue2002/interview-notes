# GD32E230 RS485 多机 IAP 升级设计（复习笔记）

芯片 GD32E230C8：Cortex-M23，片内 Flash 64 KB（`0x08000000` 起），SRAM 8 KB（`0x20000000` 起），Flash 页大小 1 KB。目标：一条 RS485 总线上，一次升级最多 128 个同型号设备，无需下载器。

## 1. Flash 分区

```text
0x08000000 ~ 0x08003BFF    Bootloader，15 KB
0x08003C00 ~ 0x08003FFF    参数/信息区，1 KB（1 页）
0x08004000 ~ 0x0800FFFF    App，48 KB
```

- Bootloader 上电先跑，代码必须压在 15 KB 以内（带 DMA+IDLE、Flash、RS485 够用）。
- 参数区正好一整页（1 KB）——这也是 Flash 最小可擦单位。改任何字段都要**擦整页再重写**，所以字段要一起管理。`dev_info_t` 才几十字节，1 KB 绰绰有余。
- App 从 `0x08004000` 开始，Bootloader 校验后跳转。Bootloader 擦写只能碰 App 区，不能擦到自身和参数区代码页。

## 2. 参数/信息区（0x08003C00）

存放本机的持久信息，掉电不丢。用来做「是否升级、跳不跳 App、身份是谁」的判断依据。

```c
#define DEV_INFO_ADDR   0x08003C00U
#define APP_BASE_ADDR   0x08004000U
#define FLASH_END_ADDR  0x08010000U

#define INFO_MAGIC      0x41505049U   // "IPPA"，标识信息区有效

#define STATE_EMPTY     0xFFFFFFFFU
#define STATE_UPDATING  0x11111111U   // 正在升级，未完成
#define STATE_VALID     0x22222222U   // 固件完整可跳转
#define STATE_INVALID   0x33333333U   // 校验失败

typedef struct {
    uint32_t magic;         // INFO_MAGIC，判断信息区有没有被初始化
    uint32_t state;         // 固件是否完整：EMPTY/UPDATING/VALID/INVALID
    uint32_t version;       // 当前固件版本号
    uint8_t  rs485_id;      // 本机 RS485 地址（1~128），出厂写入
    uint8_t  hw_version;    // 硬件版本
    uint16_t rsv0;
    uint32_t app_size;      // 固件大小
    uint32_t app_crc32;     // 整包 CRC
    uint32_t last_update;   // 上次更新时间
    uint32_t rsv[6];
} dev_info_t;
```

要点：`state` 管固件完整性（掉电保护的关键），`version` 管「要不要升级」，`rs485_id` 是本机在总线上的身份。

## 3. App 工程需要改的地方

App 不能再从 `0x08000000` 链接，要从 `0x08004000` 开始。

Keil 里改 IROM：

```text
IROM1 Start: 0x08004000
IROM1 Size : 0x0000C000
```

`system_gd32e23x.c` 改向量表偏移：

```c
#define VECT_TAB_OFFSET  ((uint32_t)0x4000)   // SCB->VTOR = 0x08000000 | 0x4000 = 0x08004000
```

## 4. 通信协议

半双工总线，上位机是主机，128 个从机平时只听不说，只有被点名才回话——这是不冲突的前提。包里加 `target_id` 做寻址：

```text
magic      2 字节    固定 0x55AA
cmd        1 字节    PING / START / DATA / END / QUERY
target_id  1 字节    0=广播(所有设备收)，1~128=单播(指定设备)
seq        2 字节    包序号
addr       4 字节    本包写入的 Flash 地址
len        2 字节    payload 长度，DATA 一般 512，START/END/PING 为 0
crc16      2 字节    payload 的 CRC16
payload    N 字节    固件数据
```

地址分配：`0x00` 广播、`1~128` 设备、其余保留。

ACK/结果回复：

```text
0xA5 0x5A  id  seq_low seq_high  status  checksum
```

`status`：`0x00` 成功 / `0x01` CRC 错 / `0x02` 地址错 / `0x03` 写 Flash 错 / `0x04` 序号错 / `0x05` 命令错。

## 5. 升级触发：从 App 进 Bootloader

设备平时在跑 App。上位机要升级时，**广播连发 10 次「升级命令 + 目标版本号」**（连发 10 次是因为广播不可靠，提高命中率）。App 收到后比较版本：

- **目标版本 > 本机版本** → 需要升级：写「停在 Bootloader」握手标志 → 软复位。
- **目标版本 ≤ 本机版本** → 已是新版或更旧，忽略，不升级。

关键是复位后 Bootloader 要知道「别自动跳 App，停下等 START」，靠一个跨复位的握手标志。推荐用 **NO_INIT RAM**（一块启动代码不清零的 RAM），软复位后仍在，且不磨损 Flash：

```c
// 放到 no-init 段，软复位不清零
__attribute__((section(".noinit"))) volatile uint32_t g_boot_flag;
#define BOOT_FLAG_UPDATE   0x55AA55AAU

// App 端：收到升级命令且版本更新
g_boot_flag = BOOT_FLAG_UPDATE;
NVIC_SystemReset();               // 软复位
```

Bootloader 上电判断（决定停下还是跳 App）：

```text
读 NO_INIT RAM 的 g_boot_flag 和参数区 dev_info

if (g_boot_flag == BOOT_FLAG_UPDATE) {
    清掉 g_boot_flag
    停在 Bootloader，等 START            // App 主动请求升级
} else if (dev_info.state == VALID && app_is_valid()) {
    跳转 App                              // 正常启动
} else {
    停在 Bootloader，等 START            // App 不完整/无效，等重刷
}
```

> `state` 存在 Flash（掉电也在，管固件完整性）；握手标志用 RAM（软复位存活即可，避免每次升级都擦 Flash）。两者配合：既能掉电保护，又不磨损 Flash。

## 6. 整体升级流程（多机，核心）

整套流程是**幂等可重跑**的：失败的设备留在 Bootloader，成功的进 App，重跑一遍时已升级的设备因版本一致自动跳过，没升级的再接一次 START。

**① 设备发现**
上位机对 ID `1~128` 逐个 PING，收到 PONG 的加入「参与列表」，确定这次有几台参与。

**② 广播 START**
广播 `START`，参与设备进升级模式：写 `state = UPDATING`、擦除 App 区。

**③ 逐帧 DATA + 点名 ACK（带重试）**

```text
广播 DATA(seq)
逐个轮询参与设备的 ACK：
    全部 ACK      → 发下一帧
    有设备没 ACK  → 对该帧重试一次
        重试后仍有没 ACK 的 → 记下，继续下一帧（不卡整体）
```

某台漏了一帧也不会连锁出错——每帧都带自己的 `addr`，它仍能正确写后续帧，只是留个洞，最后 END 的整包 CRC 会抓出来。

**④ 广播 END + 收结果**
广播 `END`，每台自算整个 App 的 CRC，写 `state = VALID / INVALID`。上位机逐个 QUERY 参与设备的结果：

```text
全部 VALID      → 本轮全部成功
仍有 INVALID/无响应 → 这些设备没成功，仍留在 Bootloader
```

**⑤ 成功进 App，失败留 Bootloader**
`state==VALID` 的设备版本已更新、跳进新 App；失败的没有有效 App，继续停在 Bootloader 等 START。

**⑥ 重跑收敛**
再走一遍 ①~⑤：

- 已进 App 的设备：收到「10 次升级命令」时**版本一致 → 不升级**，不受影响。
- 还在 Bootloader 的设备：再次收到 START → 重来一次。

**⑦ 终止条件（必须有）**
如果某台真的坏了/掉线，会无限重跑。所以设**整体最大轮数（如 3 轮）**，超过就停下，把仍失败的 ID 列表报给操作员人工处理。

时序示意：

```text
Host  -> PING 1..128                （建参与列表）
Host  -> Broadcast START
Host  -> Broadcast DATA(seq=0)
Host  -> QUERY id=3 ; id=3 -> ACK
Host  -> QUERY id=7 ; id=7 -> (无) → 重试该帧一次
...
Host  -> Broadcast END
Host  -> QUERY 参与设备结果 → 收 VALID/INVALID
成功者跳 App，失败者留 Bootloader → 重跑
```

## 7. Bootloader 接收：DMA + IDLE

帧变长（START/END 的 payload=0，DATA=512），用 **DMA + 串口 IDLE 中断**收整包：DMA 搬进缓冲区，总线空闲触发 IDLE，一包收完置标志，主循环再处理。适配变长、CPU 不死等。缓冲区要大于最大包长（约 600），设 1024。收到包先看 `target_id` 是不是广播或本机 ID，不是就丢弃。

```c
#define RX_BUF_SIZE  1024
static uint8_t  rx_buf[RX_BUF_SIZE];
static volatile uint8_t  frame_ready = 0;
static volatile uint16_t frame_len   = 0;

void USART0_IRQHandler(void)
{
    if (usart_interrupt_flag_get(USART0, USART_INT_FLAG_IDLE)) {
        usart_data_read(USART0);                 // 读 DATA 清 IDLE 标志（GD32 时序）
        dma_channel_disable(DMA0, DMA_CH);
        frame_len   = RX_BUF_SIZE - dma_transfer_number_get(DMA0, DMA_CH); // 剩余计数反推长度
        frame_ready = 1;
    }
}

int main(void)
{
    systick_config();
    boot_uart_init();          // 含 DMA 接收 + 使能 USART IDLE 中断
    dma_recv_start();

    while (1) {
        if (frame_ready) {
            frame_ready = 0;

            if (parse_packet(rx_buf, frame_len, &pkt) &&
                (pkt.target_id == 0 || pkt.target_id == g_my_id)) {   // 广播或点名给我
                if (pkt.cmd == CMD_START) {
                    mark_updating();            // state = UPDATING
                    erase_app_area();
                    send_ack();
                } else if (pkt.cmd == CMD_DATA) {
                    check_seq();                // 序号连续（含重复包幂等处理）
                    check_crc();                // 本地重算 payload CRC 与收到的比对
                    check_addr();               // 地址在 App 区
                    write_flash();
                    send_ack();
                } else if (pkt.cmd == CMD_END) {
                    if (whole_crc_ok()) mark_valid();   // state = VALID
                    else                mark_invalid();
                    send_ack();                 // 回本机结果
                } else if (pkt.cmd == CMD_QUERY) {
                    send_result();              // 单播回 VALID/INVALID
                }
            }
            dma_recv_start();                   // 处理完必须重启 DMA
        }
        // 加超时：距上次收包超阈值则报错/复位，防上位机中途掉线卡死
    }
}
```

要点：IDLE 只置标志，校验/写 Flash 放主循环；每包处理完必须重启 DMA；重复包要能幂等（同 addr 写同数据或 seq 识别重复），因为 ACK 丢了会被重发。

## 8. Flash 擦除与写入

擦除只按 App 区页地址循环，绝不整片擦；写入后回读校验；地址检查拒绝 App 区外和非 4 字节对齐。

```c
static uint8_t app_addr_valid(uint32_t addr, uint32_t len)
{
    if (addr < APP_BASE_ADDR)            return 0;
    if ((addr + len) > FLASH_END_ADDR)   return 0;
    if ((addr & 0x03U) != 0U)            return 0;   // 4 字节对齐
    return 1;
}

static uint8_t flash_write(uint32_t addr, uint8_t *data, uint32_t len)
{
    if (!app_addr_valid(addr, len)) return 0;
    fmc_unlock();
    for (uint32_t i = 0; i < len; i += 4) {
        uint32_t word = 0xFFFFFFFFU;
        for (uint8_t j = 0; j < 4; j++)
            if ((i + j) < len) {
                word &= ~(0xFFUL << (j * 8));
                word |=  ((uint32_t)data[i + j] << (j * 8));
            }
        if (fmc_word_program(addr + i, word) != FMC_READY ||
            *(volatile uint32_t *)(addr + i) != word) {   // 回读校验
            fmc_lock();
            return 0;
        }
    }
    fmc_lock();
    return 1;
}
```

最后一包不足 512 字节时按实际长度写。

## 9. App 有效性判断 + 跳转 App

跳转前检查向量表前两个字：第一个字（初始 MSP）落在 SRAM，第二个字（Reset_Handler）落在 App Flash。跳转前**必须先关掉 Bootloader 用的 DMA、USART、IDLE 中断**，否则 App 会继承一个还在跑的 DMA（可能越界写坏 App 内存）。

```c
typedef void (*pFunction)(void);

static uint8_t app_is_valid(void)
{
    uint32_t sp    = *(volatile uint32_t *)APP_BASE_ADDR;
    uint32_t reset = *(volatile uint32_t *)(APP_BASE_ADDR + 4U);
    if (sp    < SRAM_BASE_ADDR || sp    > SRAM_END_ADDR)  return 0;
    if (reset < APP_BASE_ADDR  || reset >= FLASH_END_ADDR) return 0;
    return 1;
}

static void jump_to_app(void)
{
    uint32_t sp    = *(volatile uint32_t *)APP_BASE_ADDR;
    uint32_t reset = *(volatile uint32_t *)(APP_BASE_ADDR + 4U);

    __disable_irq();
    dma_channel_disable(DMA0, DMA_CH);           // 停 DMA（指向 rx_buf 的那条）
    usart_disable(USART0);                        // 关串口和 IDLE 中断
    SysTick->CTRL = 0; SysTick->LOAD = 0; SysTick->VAL = 0;
    for (uint32_t i = 0; i < 8; i++) {            // 清 NVIC 使能与挂起
        NVIC->ICER[i] = 0xFFFFFFFFU;
        NVIC->ICPR[i] = 0xFFFFFFFFU;
    }
    SCB->VTOR = APP_BASE_ADDR;                     // 向量表指向 App
    __DSB(); __ISB();
    __set_MSP(sp);                                 // 设 App 栈顶
    ((pFunction)reset)();                          // 把 App 复位入口地址当函数调用 → 跳过去
}
```

> `((pFunction)reset)();` = 把 App 的 Reset_Handler 地址强转成函数指针并调用，等价于「跳到 App 运行」。外层括号是为优先级：先转指针再调用。

**关于 rx_buf**：它是 Bootloader 私有的全局数组。跳 App 前唯一必须做的是停掉指向它的 DMA；进 App 后这块 RAM 被 App 的启动代码重新初始化占用，原缓冲区不复存在。

## 10. RS485 注意事项

- **收发方向控制（DE/RE）**：设备回 ACK 时切到发送，发完立刻切回接收，否则漏收主机下一条或造成冲突——这是 RS485 最常见的 bug。
- 每个设备 ID 唯一（存参数区，出厂写入）；`0` 是广播地址不分配给设备。
- DATA 用广播 `target_id=0`；PING、QUERY、单播补传用 `1~128`。
- 广播阶段不能让所有设备同时回复。
- 重复包要幂等；每台独立算整包 CRC（这是漏包能被 END 抓到的安全网）。
- 掉电保护：`state=UPDATING→VALID`，某台补传中途掉电，下次上电 `state` 非 VALID，不跳残缺 App，等重刷。
- Bootloader 代码控制在 15 KB 内。

## 11. 面试怎么讲

> RS485 半双工，我让上位机做广播主控。升级时先广播升级命令带目标版本号，App 比对版本，比自己新就写一个 NO_INIT RAM 标志并软复位，Bootloader 上电看到这个标志就停下等 START。然后上位机扫 1~128 建参与列表，广播 START 让设备擦 App 区，接着逐帧广播 DATA、点名轮询 ACK，某帧有设备没收到就重试一次，不行先记下继续。发完广播 END，每台自算整包 CRC 写 VALID/INVALID，上位机逐个查结果。成功的跳 App，失败的留在 Bootloader，然后整个流程重跑——已升级的设备因版本一致自动跳过，没成功的再来一次，最多重跑几轮后把仍失败的 ID 报出来。这样升级时间主要跟固件大小相关，128 台也能收敛。
